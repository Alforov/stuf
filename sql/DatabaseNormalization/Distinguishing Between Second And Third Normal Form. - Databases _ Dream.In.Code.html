Hello,

First off, I have never been on this forum before, and the rules seem to be strictly followed. I read most of the posting rules and I'm pretty sure I'm doing nothing wrong here.

My question for you: What is the difference between second and third normal form in database normalization. I understand that second normal form is to restrict your database from having any partial dependencies. Where I am specifically confused is how that differs from third normal form (3NF); all of the definitions I have read about 3NF seem to have the same restrictions.

Re: Distinguishing between second and third normal form.

Posted 03 April 2012 - 07:50 PM

3NF is derived from 2NF, or in other words, if something is 3NF then it must be 2NF.

The techincal definition is "Every non-prime attribute of R is non-transitively dependent (i.e. directly dependent) on every super key of R", which is like trying to read government legislation.

In laymans terms, in 3NF, columns in a table will be:
  • A Primary Key (whether that be a single column, or group of columns, i.e. - a compound key)
  • Any number of Foreign Keys
  • Any number of informational columns where the information relates to the primary key

For instance, the following is 2NF:
view source
print?
1--Workplace table--
2ID (PK)
3Workplace
4EmployeeName
5EmployeePersonnelNumber

Whereas the following is 3NF:
view source
print?
1--Workplace table--
2ID (PK)
3Workplace
4EmployeeID (FK -> Employee table)
5 
6--Employee table--
7ID (PK)
8EmployeeName
9EmployeePersonnelNumber
